# Capture Checklist

Store durable memory when the task produced any of the following:

- a user preference
- a project milestone or major state change
- a strategic decision
- a repeatable lesson
- a newly important person/tool/company/service note
- an incident worth revisiting
- a summary likely to save future time

## Minimum fields to include

- clear title
- created/modified timestamps when practical
- concise summary
- relevant tags
- links to related notes/projects/entities

## Good capture style

- specific title
- short summary first
- rationale where applicable
- links to related work
- update existing notes if one already fits

## Bad capture style

- vague titles like "notes" or "stuff"
- giant unstructured dumps
- storing a decision without why it was made
- leaving durable information only in raw logs
